# LIB Draft-to-Final Workflow Implementation

## Overview
Implemented a new workflow where auto-generated LIBs are saved as DRAFT, allowing inline editing, then finalized when ready. Only finalized LIBs appear in Budget Office utilization.

## New Workflow

### Step 1: Auto-Generate LIB (DRAFT)
1. User clicks "Auto-Generate from Allocations"
2. Selects fiscal year
3. Clicks "Generate LIB"
4. System creates LIB with **status = 'draft'** (not 'approved')
5. LIB displays with "DRAFT" badge

### Step 2: Edit on Display
1. User sees "Add Item" buttons below each category
2. Can add items using inline form with UACS autocomplete
3. Can add/modify items as needed
4. All changes saved to database
5. LIB remains in DRAFT status

### Step 3: Finalize LIB
1. User reviews all items
2. Clicks green "Finalize LIB" button below Grand Total
3. Confirmation dialog appears
4. System updates status to 'approved' (final)
5. LIB now shows "FINAL" badge

### Step 4: Final LIB Behavior
1. "Add Item" buttons hidden
2. Cannot add or edit items
3. Visible to Budget Office for utilization
4. Can be printed/downloaded
5. Permanent record

## Changes Made

### 1. Auto-Generate Saves as DRAFT
**File**: `pages/lib.php` - `saveAutoGeneratedLIB()` function

```javascript
formData.append('markAsFinal', '0'); // Save as DRAFT, not final
```

**Before**: Auto-generated LIBs were marked as final immediately
**After**: Auto-generated LIBs are saved as draft for editing

### 2. Finalize Button Added
**File**: `pages/lib.php` - `generateLIBView()` function

Added green "Finalize LIB" button below Grand Total:
```html
<button onclick="finalizeLIB(${lib.id})" class="px-8 py-3 bg-green-600...">
    Finalize LIB
</button>
```

**Visibility**: Only shows for draft LIBs
**Location**: Below Grand Total row, above print footer
**Style**: Large green button with checkmark icon

### 3. Finalize Function
**File**: `pages/lib.php` - JavaScript

```javascript
function finalizeLIB(libId) {
    // Shows confirmation dialog
    // Calls finalize_lib.php API
    // Refreshes display on success
}
```

**Features**:
- Confirmation dialog with warning
- API call to finalize
- Success/error handling
- Auto-refresh display

### 4. Finalize API Endpoint
**File**: `api/finalize_lib.php` (NEW)

```php
// Updates status from 'draft' to 'approved'
UPDATE line_item_budgets SET status = 'approved' WHERE id = ?
```

**Security**:
- Session validation
- LIB existence check
- Draft status verification
- Department access control

### 5. Add Item Restriction
**File**: `api/add_lib_item.php`

Restored draft-only restriction:
```php
if ($lib['status'] !== 'draft') {
    echo json_encode(['success' => false, 'message' => 'Cannot add items to a finalized LIB']);
}
```

## Database Schema

### line_item_budgets table
```sql
status ENUM('draft', 'pending_approval', 'approved', 'rejected')
```

**Status Values**:
- `draft` - Editable, not visible to budget office
- `approved` - Final, visible to budget office, read-only

## Budget Office Visibility

### Before Finalization (DRAFT)
- ❌ Not visible in budget office utilization
- ❌ Cannot create utilization entries
- ✅ Department can edit

### After Finalization (FINAL)
- ✅ Visible in budget office utilization
- ✅ Can create utilization entries
- ❌ Department cannot edit

## API Endpoints

### POST /api/finalize_lib.php
**Parameters**:
- `lib_id`: ID of the LIB to finalize

**Response**:
```json
{
    "success": true,
    "message": "LIB finalized successfully"
}
```

**Errors**:
- LIB not found
- Already finalized
- Access denied
- Not a draft

### Issue: Finalize button not showing

- Check if `showActions` is true
- Clear browser cache

### Issue: Already finalized error

- Cannot finalize twice
- Check LIB status in database
