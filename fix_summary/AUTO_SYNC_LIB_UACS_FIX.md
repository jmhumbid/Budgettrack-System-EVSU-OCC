# Auto-Sync LIB UACS Code Mapping Fix

## Problem
When generating LIB from budget allocations, the UACS Code column was empty and all items were categorized under "B. Maintenance & Other Operating Expenses" regardless of their actual category.

## Root Cause
1. The `generate_auto_lib.php` API returned items with empty `uacs_code` fields
2. The `displayAutoGeneratedItems()` function displayed items as-is without searching for UACS codes
3. UACS code search only happened in `saveAutoGeneratedLIB()` when saving, not when displaying
4. The variable name mismatch: code used `uacsCodes` but the actual variable is `UACS_CODES`

## Solution Implemented

### 1. Updated `displayAutoGeneratedItems()` Function
**File**: `pages/lib.php`

Added UACS code search logic when displaying items:
- For allocation items (non-custom), search for UACS codes based on description
- Update item with found UACS code and proper description
- Determine category from UACS code
- Display UACS code in the table (or "N/A" if not found)

```javascript
function displayAutoGeneratedItems() {
    // ... existing code ...
    
    autoGeneratedItems.forEach((item, index) => {
        // Search for UACS code if not already set (for allocation items)
        let uacsCode = item.uacs_code || '';
        let description = item.general_desc;
        let category = item.category || 'B. Maintenance & Other Operating Expenses';
        
        if (!item.is_custom && !uacsCode) {
            // Search for UACS code based on description
            const searchResults = searchUACSForDescription(description);
            uacsCode = searchResults.code || '';
            description = searchResults.name || description;
            
            // Update the item with found UACS code and proper description
            item.uacs_code = uacsCode;
            item.general_desc = description;
            
            // Determine category from UACS code
            if (uacsCode) {
                category = determineCategoryFromUACS(uacsCode);
                item.category = category;
            }
        }
        
        // Display with UACS code
        // ...
    });
}
```

### 2. Fixed `searchUACSForDescription()` Function
**File**: `pages/lib.php`

Fixed variable name and added flattening logic:
- Changed from `uacsCodes` to `UACS_CODES` (correct variable name)
- Added logic to flatten the nested `UACS_CODES` object into a single array
- Maintained keyword mapping for common allocation terms

**Keyword Mappings**:
- "Part-time" → "Honoraria - Part-time" (UACS: 5010210001) - Category A
- "Overload" → "Honoraria - Overload" (UACS: 5010210001) - Category A
- "COS" → "Labor and Wages" (UACS: 5021601000) - Category B
- "Water" → "Water Expenses" (UACS: 5020401000) - Category B
- "Electricity" → "Electricity Expenses" (UACS: 5020402000) - Category B
- "Security" → "Security Services" (UACS: 5021203000) - Category B
- "Internet" → "Internet Subscription Expenses" (UACS: 5020503000) - Category B
- "Telephone" → "Telephone Expenses" (UACS: 5020502001/5020502002) - Category B

### 3. Optimized `saveAutoGeneratedLIB()` Function
**File**: `pages/lib.php`

Simplified to use already-searched UACS codes:
- Removed redundant UACS search (already done in `displayAutoGeneratedItems()`)
- Use the UACS code and description from the item directly
- Use the category that was already determined

```javascript
function saveAutoGeneratedLIB() {
    // ... existing code ...
    
    autoGeneratedItems.forEach(item => {
        // Use the UACS code and description that were already searched
        const uacsCode = item.uacs_code || '';
        const uacsName = item.general_desc;
        const category = item.category || 'B. Maintenance & Other Operating Expenses';
        
        formData.append('category[]', category);
        formData.append('particulars[]', uacsName);
        formData.append('account_code[]', uacsCode);
        formData.append('amount[]', item.total_amount);
    });
    
    // ... save to database ...
}
```

### 4. Category Determination Logic
**File**: `pages/lib.php`

The `determineCategoryFromUACS()` function categorizes based on UACS code prefix:
- `501xxx` → "A. PERSONAL SERVICES"
- `502xxx` → "B. Maintenance & Other Operating Expenses"
- `506xxx` → "C. Capital Outlay"

## Expected Behavior

### When Generating LIB:
1. User clicks "Auto-Generate from Allocations" button
2. Selects fiscal year and clicks "Generate LIB"
3. Modal displays table with:
   - Source column (Allocation/Custom badge)
   - UACS Code column (populated with found codes)
   - Particulars column (proper UACS description)
   - Amount column
   - Actions column

### When Saving LIB:
1. User reviews the generated items with UACS codes
2. Clicks "Save LIB"
3. System saves to database with:
   - Correct categories based on UACS codes
   - Proper UACS descriptions as particulars
   - UACS codes in account_code field
   - Amounts from allocations
